# 人工付款、開通、續約與到期封鎖

## 正式流程

1. 客戶先註冊並取得 3 日線上試用，業態只能選一般企業 ERP、零售 POS 或餐飲 POS 其中一種。
2. 客戶從方案頁送出需求。系統先保存需求至平台管理後台，再嘗試寄送通知信至艾琳設計；即使 Gmail 暫時故障，需求仍不會遺失。
3. 艾琳設計聯絡客戶確認環境、方案、席次與付款方式。網站不提供直接付款連結，也不接受客戶自行把帳號改為已付款。
4. 確認銀行或付款平台確實入帳後，平台管理者在公司授權視窗填寫實收金額、付款方式、付款參考編號與實際入帳時間，勾選付款確認後才可開通。
5. 中央一次完成付款紀錄、方案、席次、到期日、啟用碼與授權事件寫入。任何一項失敗都不會留下半套開通狀態。
6. 客戶使用公司代碼與一次性啟用碼安裝公司主機及已購買數量的操作工作站。

## 日期與續約規則

- 月租從開通基準日加一個曆月；若基準日是 29、30、31 日而次月沒有該日，使用次月最後一日。
- 年租收取 10 個月定價但提供完整 12 個月使用期。
- 客戶提前續約時，從目前尚未到期的到期日繼續往後加，不會從付款當天重算而損失剩餘天數。
- 一次買斷沒有使用期限；第一年版本與 AI 維護到期後，後續維護依每年 NT$2,000 另行確認。
- 中央簽發的離線租約最長 24 小時，但不得超過實際訂閱到期時間。例：訂閱剩 2 小時，只能簽 2 小時租約。
- 撤銷、無效啟用碼、超席次或中央明確回覆到期時立即封鎖；只有暫時斷網／中央 5xx 才可沿用尚未到期且簽章有效的租約。

## 防重複與防竄改

- 同一公司不能重複使用相同付款參考編號，避免管理者重複按下開通造成重複入帳。
- 每筆付款紀錄包含方案、週期、報價、實收、方式、參考編號、入帳時間與確認管理者，中央以機密稽核密鑰建立雜湊。
- 授權事件雜湊鏈同時保存付款紀錄 ID 與付款雜湊。管理後台的「驗證防竄改紀錄」會同時檢查付款、授權與業務操作三類紀錄。
- 客戶端只取得中央簽章結果，不取得付款紀錄密鑰、授權私鑰或可自行修改方案的 API 權限。

## 通知設定

Vercel 本身不提供信箱。正式環境須設定 `GMAIL_USER`、`GMAIL_APP_PASSWORD` 與 `CONTACT_TO_EMAIL`，或日後換成具重送、退信與網域驗證的商業寄信服務。通知信只是提醒；平台管理後台保存的待聯絡需求才是工作清單。
