# POS 硬體採購與實機驗收表

目前自動測試只證明艾琳 ERP／POS 的協定資料、錯誤分支、作業系統印表機偵測與第二螢幕客顯流程可運作。下列項目必須填入實際型號、驅動與收單環境後逐台簽收，未完成前不得宣稱該型號已相容。

## 1. 條碼槍

記錄廠牌、型號、韌體、USB／Bluetooth、HID 設定、前綴、後綴與鍵盤語系。

- 連續掃描 EAN-8、EAN-13、UPC-A、Code 128 與客戶自訂內碼各 20 次，不得漏碼、黏碼或多輸入字元。
- Enter／CR／LF 結束字元能正確移除；焦點不在掃碼欄時，按 F2 可回到掃碼欄。
- 有效 GTIN 顯示檢查碼通過；自訂內碼即使不是 GTIN，也只能精確命中一個商品。
- 不存在、停用、重複條碼與庫存不足都要阻擋結帳並顯示可理解訊息。
- 休眠喚醒、拔插、藍牙斷線及快速連掃後能復原。

## 2. 80mm 熱感印表機

記錄廠牌、型號、連線方式、序號、韌體、作業系統驅動、紙寬、DPI、切刀及字碼設定。

- 桌面硬體診斷能在作業系統印表機清單找到正確裝置。
- 列印 80mm 測試頁，核對繁體中文、英數、長商品名、金額對齊、換行、濃度與不截字。
- 連續列印至少 30 張交易收據與 10 張補印收據，單號不得錯置或重複。
- 自動切紙連續 50 次；缺紙、上蓋開啟、離線與重連時不得把失敗誤標成已列印。
- 網路型印表機重開機或 IP 變更後，操作頁能指出離線並保留補印方式。
- 正式電子發票證明聯須另依電子發票平台、QR Code 與法規格式驗收，交易收據不可冒充發票。

## 3. 錢櫃

記錄廠牌、型號、RJ11／RJ12 接法、額定電壓、印表機錢櫃埠規格、開櫃接腳與脈衝時間。不得在未確認 12V／24V 與接線相容前通電測試。

- 僅現金結帳、已核准的投入／提出及具權限的測試動作可送出開櫃命令。
- 一筆交易只開櫃一次；網路重試與頁面重新整理不得重複脈衝。
- 連續開櫃 50 次，鎖具、彈簧與脈衝均正常。
- 主管核准、申請人、原因、金額及時間須留在稽核紀錄。
- 若型號有開／關狀態感測器，結班前須測試未關櫃警示；無感測器需標示此限制。

## 4. 客戶顯示器

記錄廠牌、型號、解析度、HDMI／DisplayPort／USB 顯示介面及螢幕編號。現行通用方案是作業系統第二螢幕；傳統 VFD／序列埠客顯需要型號專用介接器。

- 艾琳 ERP App 能偵測第二螢幕並在該螢幕全螢幕開啟客顯。
- 掃描、改數量、折扣、付款、找零與交易完成訊息在 1 秒內同步。
- 關閉再開啟客顯、拔插螢幕及重啟 App 後能重新連線。
- 1920×1080、1366×768 與實際採購解析度不得溢出或遮字。
- 客顯不得顯示管理功能、成本、會員敏感資料或其他公司的資料。

## 5. 刷卡機／金流終端機

記錄收單銀行／支付商、終端機型號、ECR 介接版本、連線方式、商店代號、端末代號及測試環境。僅模擬核准不等於能正式收款。

- 完成收單商測試環境的核准、拒絕、取消、逾時、沖正、退貨與日結批次。
- POS 金額與終端機金額必須一致，收銀員不可在授權後任意改回較小金額。
- 每筆使用唯一請求碼；逾時須用同一碼查詢，不得直接重刷造成重複扣款。
- 核准碼、交易序號與遮罩卡號可留存；完整卡號、磁軌與安全碼不得進入 ERP 資料庫或日誌。
- 斷網、終端機重啟、銀行回覆延遲與 POS 回覆遺失均須驗證「交易結果不明」復原流程。
- 實際新台幣 1 元或收單商指定金額完成端對端交易、沖正及退款後，才可簽署正式上線。

## 驗收結論

每一設備只能標示以下一種狀態：

- `SIMULATION_PASS`：軟體模擬通過，尚未驗證實機。
- `DRIVER_DETECTED`：作業系統已偵測，尚未完成營業流程。
- `MODEL_ACCEPTED`：指定型號在指定 OS、驅動與連線方式全部通過。
- `REJECTED`：任一必要項失敗，不得交付客戶使用。

更換型號、韌體、主要驅動、作業系統大版本、收單商或電子發票介接時，必須重新驗收。
