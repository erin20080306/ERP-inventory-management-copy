# ERP／POS 完整度稽核與改善路線

更新日期：2026-07-16

> 本文件是依目前原始碼、資料庫結構與本機操作測試做的產品稽核，不等同會計、稅務、資安或電子發票認證。

## 產品架構結論

不應把一般企業 ERP 與 POS 做成完全相同的一張畫面。正確方向是：

- 共用同一套商品、客戶、供應商、庫存、付款、會計、權限、稽核與報表資料。
- ERP 工作台以單據、審核、過帳、勾稽及報表為主。
- POS 工作台以條碼、會員、促銷、快速付款、收據、退換貨與開結班為主。
- 公司開通時在「企業 ERP、零售 POS、餐飲 POS」三種互斥業態中選一種；POS 版仍共用進銷存與會計資料，但前台、後台權限分開。

鼎新公開資料也採「POS 前台 + 總部管理 + ERP 整合」架構，而不是把 ERP 表單原封不動放進收銀畫面。功能與資料流程可做到同級，但不應逐像素複製其專有畫面、文字或程式。

## 2026-07-16 第一批已改善

1. 新增 POS 專用商品查詢 API，掃碼可搜尋完整商品主檔，不再受前 200 筆限制。
2. 條碼或貨號重複時拒絕模糊結帳，商品新增／修改也會阻擋重複條碼。
3. 新增會員／客戶搜尋與指定，結帳會寫回原有客戶及帳務資料。
4. 新增現金、刷卡、行動支付、轉帳及最多四種混合付款，支援交易參考號碼。
5. 新增逐項折扣輸入，後端仍會重新驗算，不信任瀏覽器金額。
6. 新增 80mm 交易收據列印與重印入口，並明確標示「非電子發票證明聯」。
7. 新增收銀台管理，可指定門市倉庫、啟用／停用並保留歷史交易。
8. 新增 F2 返回掃描框，支援一般 USB／藍牙 HID 條碼槍的鍵盤輸入模式。

## 2026-07-16 第二批已改善

1. 錢櫃投入、提出與營業中抽離改為申請／核准紀錄，只有核准金額會影響結班應有現金；待核申請會阻擋結班。
2. 新增最多 20 筆伺服器暫存交易、一次性取回鎖，以及每 700ms 寫入資料庫的停電復原草稿；暫存與草稿都會在取回時重查價格及庫存。
3. 建立電子發票 outbox 狀態機、事件鎖、失敗退避重送、管理頁與 cron 入口；本機只產生 `TEST-` 號碼並標示不可報稅。
4. 新增條碼 HID、80mm ESC/POS、錢櫃 ESC p、雙分頁客顯及刷卡核准／拒絕／逾時的無實機診斷頁與自動測試。
5. 退貨可逐項指定良品、瑕疵或報廢；只有良品進入指定門市的可售庫存，並分開記錄不回庫成本。
6. 換貨先建立退款，再將下一筆新銷售與退款單一對一綁定；跨店退貨會記錄實際受理倉庫。
7. 新增會員點數賺取、折抵、退款按比例回沖，且用資料庫鎖避免兩台收銀機重複使用同一餘額。
8. 新增自動促銷擇優、優惠券總次數／每會員限制，以及手動折扣購物車指紋；購物車變更後原主管核准立即失效。

### 第二批本機驗收結果

- Production build、TypeScript、Prisma schema 與 12 筆資料庫遷移通過。
- 整合測試通過：部分／全額退款、原付款方式分配、瑕疵不回庫、跨店良品入庫、點數回沖、傳票平衡及失敗原子回復。
- 瀏覽器實測通過：錢櫃申請核准、伺服器暫存取回、重新載入出現停電草稿、測試電子發票開立與佇列、硬體模擬、客顯跨分頁同步、促銷＋券＋點數結帳、瑕疵換貨與新單連結。
- 無實機測試只能證明軟體協定與錯誤流程；USB／網路驅動、中文字碼、切紙、錢櫃電壓、刷卡收單與發票機仍須指定型號實測。

## 2026-07-16 第三批已改善

1. 產品版別拆成企業 ERP、零售 POS、餐飲 POS；零售帳號不可進餐飲網址，企業帳號不可進 POS，前端入口與 API 同時檢查。
2. 新增工作區選擇頁，把 POS 前台、進銷存後台與會計後台分開顯示；角色可分別授予 POS、餐飲、庫存與會計權限。
3. 新增餐飲圖片點餐、桌位、桌單、送廚、廚房製作／待出／已出餐及結帳流程；未結餐桌會阻擋關班。
4. 餐飲結帳沿用既有 POS 原子交易，同步扣庫存、建立已過帳銷售單、收款及兩張平衡傳票。
5. 餐飲結帳新增紙本、手機載具、自然人憑證、捐贈碼與公司統編欄位，但仍清楚標示為電子發票模擬佇列。
6. 共用 ERP 表格改成游標先跳格、背景儲存；實測方向鍵約 15ms 完成焦點移動，不等待伺服器回應。
7. 桌面工作站加入公司代碼自動連線：中央以公司代碼＋啟用碼查詢，回傳 10 分鐘有效的 Ed25519 簽章主機網址與 CA 憑證；查詢不占席次，租約成功後才綁定工作站。
7. 新增 Excel 多列多欄貼上、儲存格／整列複製、右鍵編輯／刪除，以及表頭右鍵新增自訂欄位、隱藏及恢復欄位。
8. 商品圖片可由有商品修改權限的管理者上傳／更換，餐飲點餐即時讀取；本機 Docker 新增持久化圖片 volume，並記錄上傳稽核。
9. 安裝腳本支援三種業態，管理員下載頁會依 Windows x64、Windows ARM64 與 macOS Apple Silicon 標示安裝包。

### 第三批本機驗收結果

- Production build（116 routes）、TypeScript 與桌面安全設定測試通過。
- 餐飲實際交易通過：桌位開桌、圖片點餐、KDS 製作、出餐、現金結帳、桌位釋放與庫存 100→99。
- 資料庫核對通過：銷售單 `POSTED`、現金收款 220、POS 銷售傳票借貸各 300、POS 收款傳票借貸各 220。
- 關班防呆通過：存在一桌未結餐單時回傳 HTTP 409；取消測試桌單後桌位恢復。
- 表格瀏覽器驗收通過：上下鍵 15ms 跳格、Excel 兩列貼上、右鍵刪除入口、表頭新增／隱藏／恢復欄位。
- 圖片瀏覽器驗收通過：上傳 PNG 後餐飲商品圖片 `src` 同步更新；測試後還原示範圖片。
- 產出內部測試包：macOS ARM64 DMG／ZIP、Windows x64／ARM64 EXE；未簽章／未公證，不可直接宣稱正式商用發行版。

## 後續必要階段

### P0：正式交易安全

- 把自訂欄位定義與欄位值由瀏覽器 localStorage 移到租戶資料庫，才能跨工作站同步及納入權限、備份與稽核。
- 為桌面客戶端建立真正的本機交易 outbox、重送與衝突策略；目前 24 小時授權暫離線不等於交易可完全離線。
- 自動化兩台工作站同時結帳、退款、開桌及移桌的競態測試。
- 完成安裝包簽章、公證、自動更新與版本回滾。

### P0：電子發票正式介接

- 選定財政部 Turnkey 或合格加值服務中心，取得測試與正式憑證。
- 建立字軌配號鎖、發票狀態機及不可重複取號交易。
- 支援 B2C／B2B、手機條碼、自然人憑證、捐贈碼、紙本證明聯。
- 支援開立、作廢、折讓、上傳、查詢、失敗補送、空白未使用字軌與日結對帳。
- 發票列印須依最新格式、QR Code／Barcode、隨機碼與紙張規定驗收。

### P1：專業門市能力

- 20 組伺服器暫存單、預購／訂金、商品指定配送、跨店取貨。
- 會員點數、等級、優惠券、滿額／滿件／組合／會員價與促銷優先順序。
- 換貨、瑕疵不回庫、跨店退貨、禮券／儲值金與混合退款。
- 相機 1D／2D 掃碼、條碼標籤、秤重條碼、客顯、錢櫃、刷卡機、發票機與印表機健康檢查。
- 門市／收銀員／時段／品類報表、客單價、連帶率、毛利、差異與促銷成效。

### P1：ERP 內控與完整流程

- 請購 → 詢價 → 比價 → 採購 → 驗收 → 進貨發票 → 應付的完整勾稽。
- 報價 → 訂單 → 信用檢查 → 揀貨 → 出貨 → 發票 → 應收 → 收款的完整勾稽。
- 盤點、調撥、批號／效期／序號、成本月結及不可回寫已關帳期間。
- 會計分錄來源追溯、反向傳票、銀行對帳、稅務申報與期末關帳清單。

## 驗收標準

- 不以「畫面能開」作為完成；每條流程都要驗證資料、庫存、付款、傳票、稅額及稽核紀錄一致。
- 條碼槍、印表機、錢櫃、客顯及金流設備必須用指定型號實機驗收。
- 電子發票必須通過測試環境、錯誤重送與財稅規格驗收後，才可標示為可正式開立。
- 斷線、連點、重送、兩台電腦同時結帳／退款、停電重啟都必須有自動測試或演練紀錄。
